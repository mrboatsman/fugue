1//! Subsonic API proxy layer.
2//!
3//! This module implements the core proxy that sits between Subsonic clients
4//! and multiple Navidrome backends.
5//!
6//! # Fan-Out Pattern
7//!
8//! For browsing and search requests, Fugue sends the request to all backends
9//! in parallel and merges the results. This is handled by the [`fanout`]
10//! module — responses are collected concurrently via `tokio::join` and
11//! combined into a single Subsonic response.
12//!
13//! # ID Namespacing
14//!
15//! Every item ID returned to clients is an opaque, namespaced token that
16//! encodes both the backend index and the original ID (see [`crate::id`]).
17//! When a client sends an ID back (e.g. to stream a song), Fugue decodes
18//! it to determine which backend to route the request to. Clients never
19//! see raw backend IDs.
20//!
21//! # Stream Proxying
22//!
23//! Audio streaming, downloads, and cover art are proxied via zero-buffer
24//! passthrough — bytes flow directly from the backend HTTP response to the
25//! client response with no intermediate buffering. Transcoding parameters
26//! (`maxBitRate`, `format`) are forwarded transparently to the backend.
27//!
28//! # Submodules
29//!
30//! - [`backend`] — `BackendClient` for making authenticated Subsonic API calls
31//! - [`fanout`] — parallel request dispatch and response merging
32//! - [`router`] — axum router wiring for all Subsonic endpoints
33//! - [`stream`] — streaming proxy (audio, downloads, cover art)
34
